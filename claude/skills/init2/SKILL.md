# init2 — AGENTS.md / CLAUDE.md ジェネレーター

プロジェクトルールを **AGENTS.md**（AI共通ルール）と **CLAUDE.md**（Claude固有設定）の2ファイルに分けて管理する。CLAUDE.md は AGENTS.md を `@AGENTS.md` でインポートするため、共通ルールの重複を避けられる。

## フェーズ1: プロジェクト分析

まずプロジェクトのコンテキストを把握する。以下を調べる（ファイルを読む前に推測しない）：

1. `ls`, `find`, `git log --oneline -10` でプロジェクト構造を確認
2. `package.json` / `go.mod` / `Cargo.toml` / `pyproject.toml` 等でスタック特定
3. 既存の `AGENTS.md` / `CLAUDE.md` / `README.md` があれば読む
4. `.github/workflows/` があればCI/CDパイプラインを把握

不明点はユーザーに聞く。特に以下は必ず確認する：

- このプロジェクトでAIに何をしてほしいか（コーディング支援 / コードレビュー / リファクタリング など）
- 特に守ってほしいルールや禁止事項はあるか
- Claude固有の設定（ツール制御・応答トーン等）があるか

## フェーズ2: AGENTS.md 生成

AI共通ルールを AGENTS.md に書く。Claude Code 以外のAIツール（GitHub Copilot、Cursorなど）でも読み込まれることを想定し、ツール固有の記述は含めない。

## フェーズ3: CLAUDE.md 生成

CLAUDE.md は `@AGENTS.md` で共通ルールをインポートし、Claude固有の設定のみを追記する。Claude固有の設定がない場合は、インポート行だけのファイルでよい。

---

## CLAUDE.md テンプレート

```markdown
@AGENTS.md

[以下はClaude固有の設定がある場合のみ追記する]

## ツール使用方針

[Claude Code のツール使用をどう制御するか]

独立した操作は並列実行してよい（複数ファイルの読み込み、並行検索など）。 ただし以下は必ずユーザーに確認してから実行する：

- `git push --force` / `git reset --hard`
- ファイルの削除
- 本番環境への変更
- 外部サービスへのメッセージ送信

## 応答スタイル

[Claude の応答トーンや形式を指定する場合]

- 回答は日本語で行う
- 説明は簡潔に。コードが自明な場合はコメントを省く
```

## セクション別ガイドライン

### AGENTS.md と CLAUDE.md の使い分け

| 書く場所      | 内容                                      |
|-----------|-----------------------------------------|
| AGENTS.md | プロジェクト概要・技術スタック・コーディングルール・コマンド・ディレクトリ構成 |
| CLAUDE.md | Claude Code のツール制御・応答スタイル・Claude固有の禁止事項 |

### 指示の書き方原則

**良い例（なぜかを添える）:**

```
ellipsis（...）を使わない。
理由: このプロジェクトの出力は音声読み上げエンジンに渡されるため、
ellipsis の読み方が未定義で不自然な読み上げになる。
```

**避けるべき例:**

```
NEVER use ellipses
```

後者も機能するが、前者の方がAIが文脈を理解して例外ケースを適切に判断できる。

### 出力フォーマットの指定

「〜するな」より「〜せよ」の形式で書く：

| 避ける           | 推奨                                   |
|---------------|--------------------------------------|
| マークダウンを使うな    | 回答は plain text の段落で記述せよ              |
| 長い回答はするな      | 要点を3行以内で答えよ                          |
| コードブロックを多用するな | コードは \`\`\` ブロックのみに限定し、説明は prose で書け |

### ツール使用の制御（CLAUDE.md 専用）

積極的に動いてほしい場合:

```
意図が明確なら実装まで進めてよい。
不明点はツールで調べて解決し、推測で止まらない。
```

慎重に動いてほしい場合:

```
変更前に必ず確認を取る。
ファイル編集・実行・削除は明示的な指示があってから行う。
```

### エージェント安全性

不可逆な操作（削除、force push、本番変更）は明示的にリストアップしておく。 Claude はリストにある操作を見たとき、自動的に一時停止してユーザーに確認する。

## 生成後の確認

両ファイルを生成したら、ユーザーに以下を確認する：

1. AGENTS.md に不足しているルールや文脈はないか
2. Claude固有の設定で追加が必要なものはないか
3. 逆に不要なセクションはないか
4. プロジェクト固有の禁止事項（触れてはいけないファイル、変えてはいけない設定など）があれば追加する
